Why Tailwind CSS Isn't for Me

‒ Views:



Published on: November 6, 2024

Tailwind's creators are clearly talented, but it's just not for me. I tried using it, but it felt too complicated. I find plain CSS way simpler and easier to work with.

Honestly, I have some real concerns about Tailwind, and what frustrates me is when I bring them up, some people just call me an idiot. As someone with a lot of experience in web development, that doesn't feel right. I may not be the best, but I love the web and I value simplicity over complicated frameworks and all of this stuff.

#1 You Have to Learn a Different Way to Write CSS


Like Bootstrap, Tailwind uses short and awkward class names. This means you'll need to learn its specific syntax before you can use it easily. When you first start, it can feel slow, and you'll find yourself constantly looking up which classes to use. So, the promise of "fast development" doesn't really kick in until you get familiar with it and honestly, that's harder than just using regular CSS.

#2 Tailwind makes HTML look ugly and Difficult to Read.


A lot of people don't like Tailwind because it makes HTML harder to read. I agree with that point because, if you care about clean and readable code, it can make things harder to follow.


Here's an example of Tailwind CSS code :


<div class="flex items-center justify-between p-4 bg-gray-200 rounded-lg shadow-lg">
<h1 class="text-lg font-bold text-blue-600">Hi!</h1>
<button class="bg-blue-500 hover:bg-blue-700 text-white font-bold py-2 px-4 rounded">Click Me</button>
<p class="text-sm text-gray-600 mt-2">This is a simple description.</p>
</div>

If you think this code doesn't look that bad, I suggest you check out Aleksandr Hovhannisyan's blog post too!


I know some of you are probably saying, "Just use @apply to keep your HTML clean!" And yes, that's a possible solution. In fact, that's what I've done.


#3 @apply is not a standard solution and has some issues.


Some Tailwind fans don't agree with me, so let me explain it simply:

  • Using @apply p-4 in a CSS file only works if you have Tailwind set up in your project. If you remove Tailwind, that CSS won't work, and your site will break.

  • Tailwind-specific CSS is non-standard and doesn't work well with other frameworks or tools. Once you start with Tailwind, you're stuck with it.

  • Also, when you write @apply everywhere in your CSS, you're not really learning CSS. You're writing Tailwind-specific code. For example, writing @apply grid isn't the same as learning to use display: grid; in plain CSS.

＠ Final Thoughts


If you like Tailwind, go ahead and use it! But please don't try to convince me it's the future of web design. It works for some, but it's not the solution for everyone. Everyone has their own preferences, and that's okay. Just remember, there are other ways to write clean and maintainable CSS too.
